# Project Sharding Strategy

As the Second Braincell Vault system evolves, managing modularity across multiple projects becomes essential. This file outlines the plan for splitting the Vault into manageable subprojects, each with a clear domain.

---

## 🧩 Project Shard Plan

### 🧠 Second_Braincell_Core
- Primary logic systems
- Narrator files
- HUD and structural references
- Daily Log templates
- Cue protocols

### 📝 Second_Braincell_Logs
- System_Changelog.md
- Vault_Worktime_Tracker.md
- Thread_Link_Tracker.md
- RealTime_Simulation logs
- Event-based anomaly logs (future)

### 🩺 Second_Braincell_Health
- EDS symptom tracking
- Passive discomfort cueing
- Health journaling tools
- Watchlist-style logic (GI, RLS, joint instability)

### 📅 Second_Braincell_Events
- Countdowns
- Trip checklists
- Travel Mode logic
- Event memory/lore

### 🔮 Second_Braincell_Chaos_Expansion (Future)
- Experimental narrator packs
- Alternative Vault metaphors
- Liminal systems
- Lore-based overlays and SCP mimicry

---

## ✅ Benefits

- Stay under 20-file Project limit
- Clear separation of responsibilities
- Easier to swap/overwrite individual vaults
- Keeps narrative modes clean per system
